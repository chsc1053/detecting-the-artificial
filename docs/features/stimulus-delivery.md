# Stimulus Delivery

## Implemented behavior

- **Text:** Inline `text_content` from the API; rendered in `StimulusView`.
- **Image / video / audio:** Media delivery is implemented through the current storage/upload path and is rendered in participant UI components (`<img>`, `<video>`, `<audio>`), including forced-choice and single-item flows.

## Notes for deployed environments

- **S3 upload permissions (server side):** presigned uploads are generated by the API, so the API's EC2/Elastic Beanstalk instance role must allow at least `s3:PutObject` on `arn:aws:s3:::<bucket>/stimuli/*` (plus list/get/delete as needed).
- **CORS (browser side):** S3 bucket CORS must allow the exact admin origin used by the web environment (scheme + host + port).
- **Clipboard warning is usually HTTP-only:** if admin is served over `http://...elasticbeanstalk.com`, browsers can block clipboard API calls. The upload can still succeed even when auto-copy fails; manual copy works.
