# Stimulus Delivery

## Implemented behavior

- **Text:** Inline `text_content` from the API; rendered in `StimulusView`.
- **Image / video / audio:** Media delivery is implemented through the current storage/upload path and is rendered in participant UI components (`<img>`, `<video>`, `<audio>`), including forced-choice and single-item flows.
